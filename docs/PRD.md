# SampleOrderSystem — 제품 요구사항 문서 (PRD)

**버전:** 1.0  
**작성일:** 2026-06-12  
**프로젝트:** 반도체 시료 생산주문관리 과제 (S-Semi)  
**기술 스택:** Java 17+, Gradle, JUnit 5

---

## 1. 과제 개요

### 1.1. 배경

여기 가상의 반도체 회사 "S-Semi" 가 있습니다.
이 회사는 다양한 종류의 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게
납품하고 있습니다.
시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고됩니다.
그런데 최근 들어 주문량이 급증하면서 문제가 생겼습니다.

"어, 이 주문 처리됐나요?"\
"공정 예약을 했는데, 언제 완성되는지 모르겠어요."\
"이미 충분한 시료 재고가 있는데, 왜 추가 공정이 돌아가고 있나요?“

엑셀과 메모장으로 주문을 관리하다 보니, 실수가 잦고 재고와 공정 현황을 한눈에 파악하기 어려웠습니다.
이러한 이유로 S-Semi에서는 더 체계적인 시료 관리를 위한 "반도체 시료 생산주문관리 시스템" 을 개발하기로 결정
했습니다.

---

### 1.2. 역할별 흐름도

'고객'은 '주문 담당자'에게 시료를 요청한다.\
'주문 담당자'는 '생산 담당자'에게 주문서를 전달한다.\
'생산 담당자'는 '주문 담당자'에게 주문을 승인하거나 거절한다.

| 역할     | 업무     | 설명                         |
|--------|--------|----------------------------|
| 고객     | 시료 요청자 | 필요한 시료를 이메일로 요청            |
| 주문 담당자 | 주문서 관리 | 요청에 맞게 주문서 작성              |
| 생산 담당자 | 시료 생산 및 승인 | 개발 시료 등록, 주문 수신 후 승인 또는 거절 |

## 2. 시스템 개요
### 2.1. 주문 흐름
- '주문 담당자'가 주문을 등록하면 'RESERVED' 상태가 된다.
- '생산 담당자'는 주문을 거절하거나 승인할 수 있다.
  주문을 거절하면 'REJECTED' 상태가 된다.
  주문을 승인하면 재고를 확인한다.
  재고가 부족한 경우 '생산 라인'에 생산 요청을 하고 'PRODUCING' 상태가 된다.
  재고가 충분하거나 'PRODUCING' 상태에서 생산이 완료되었다면 출고 준비가 가능한 'CONFIRMED' 상태가 된다.
- '생산 담당자'는 'CONFIRMED' 상태에서 출고 처리를 할 수 있고, 이 경우 'RELEASE' 상태가 된다.
- 'RELEASE' 상태가 되면 시료가 고객에게 도달한다.

### 2.2. 생산 라인
- 공장에서 시료 하나를 생산하는 설비 흐름
- 하나의 생산 라인은 시료를 하나씩 생산
- 주문이 들어온 시료에 대해서만 생산

### 2.3. 시스템 동작 방식
- 시스템은 콘솔 기반으로 동작한다.
- 담당자가 직접 명령을 입력해서 시료를 등록하고 주문을 처리한다.

### 2.4. 데이터 영속성
- 시스템 종료 후에도 데이터를 유지한다.
- 시료, 주문, 생산 큐 상태를 `data/db.json`에 JSON 형태로 저장한다.
- 시스템 시작 시 자동으로 불러오며, 각 데이터 변경(시료 등록·주문 등록·승인·거절·출고·생산 완료) 성공 직후 즉시 자동 저장한다.

## 3. 주문 상태 흐름
- 모든 주문은 아래의 상태 중 하나를 보유
- REJECTED는 거절된 주문으로 정상 흐름 외의 상태이며 모니터링에서 제외

### 3.1 상태 정의

| 상태 | 설명    | 진입 조건                                             |
|---|-------|---------------------------------------------------|
| `RESERVED` | 주문 접수 | 주문 담당자가 주문 등록 시 |
| `REJECTED` | 주문 거절 | `RESERVED` → 생산 담당자 거절 |
| `PRODUCING` | 주문 승인 완료 및 재고 부족으로 생산 중 | `RESERVED` → 생산 담당자 승인 → 재고 부족 |
| `CONFIRMED` | 주문 승인 완료 및 출고 대기 중 | `RESERVED` → 생산 담당자 승인 → 재고 충분 또는 `PRODUCING` → 생산 완료 |
| `RELEASE` | 출고 완료 | `CONFIRMED` → 생산 담당자 출고 처리 |

### 3.2 상태 전이 다이어그램

```
[주문 담당자 주문 등록]
         │
         ▼
      RESERVED
      /       \
[거절]/         \[승인]
    ▼             ▼
REJECTED     재고 확인
             /       \
   [재고 부족]/         \[재고 충분]
            ▼           ▼
        PRODUCING ──► CONFIRMED
        [생산 완료]        │
                    [출고 처리]
                         │
                         ▼
                      RELEASE
```

### 3.3 허용된 상태 전이 매트릭스

> 매트릭스에 없는 전이를 시도하면 `InvalidOrderStateTransitionException`을 던져야 한다.

| 현재 상태 \ 다음 상태 | REJECTED | PRODUCING | CONFIRMED |  RELEASE  |
|---|:---:|:---:|:---:|:---------:|
| `RESERVED` | O (거절) | O (승인 + 재고 부족) | O (승인 + 재고 충분) |     -     |
| `PRODUCING` | - | - | O (생산 완료) |     -     |
| `CONFIRMED` | - | - | - | O (출고 처리) |
| `REJECTED` | - | - | - |     -     |
| `RELEASE` | - | - | - |     -     |

**전이 주체 요약**

| 전이 | 주체 | 조건 |
|---|---|---|
| RESERVED → REJECTED | 생산 담당자 | 주문 거절 |
| RESERVED → PRODUCING | 생산 담당자 | 승인 시 재고 부족 |
| RESERVED → CONFIRMED | 생산 담당자 | 승인 시 재고 충분 |
| PRODUCING → CONFIRMED | 시스템 (생산 라인 완료) | 생산 완료 이벤트 |
| CONFIRMED → RELEASE | 생산 담당자 | 출고 처리 |

### 4.2 시료 관리

시료(Sample)는 시스템의 가장 기본 단위이다. 시스템에 등록된 시료만 주문할 수 있다.

**시료 속성**

| 속성 | 형식 | 설명 |
|---|---|---|
| 시료 ID | `S-XXX` (XXX: 000부터 시작하는 세 자리 숫자) | 시스템 내 고유 식별자 |
| 시료명 | 문자열 | 시료의 이름 |
| 평균 생산시간 | 0 초과의 실수 (단위: min/ea) | 시료 1개를 생산하는 데 걸리는 평균 시간 (소수점 허용) |
| 수율 | 0 초과 1 이하의 소수 | 정상 시료 수 / 총 생산 시료 수 |
| 재고 수량 | 0 이상의 정수 | 현재 출고 가능한 시료 수 |

#### 4.2.1 시료 등록

| 항목 | 내용 |
|---|---|
| 입력 | 시료 ID, 시료명, 평균 생산시간, 수율 |
| 처리 결과 | 시료를 시스템에 등록 (초기 재고 수량은 항상 0으로 고정) |
| 출력 | 등록된 시료 정보 요약 및 완료 메시지 |
| 예외 | 동일 시료 ID 이미 존재 → 오류 메시지 출력 후 시료 ID 재입력 요청 (다른 필드는 묻지 않음) |
| 예외 | 시료명이 공백 → 오류 메시지 출력 후 재입력 요청 |
| 예외 | 평균 생산시간이 0 이하 → 오류 메시지 출력 |
| 예외 | 수율이 0 이하 또는 1 초과 → 오류 메시지 출력 |

#### 4.2.2 시료 목록 조회

| 항목 | 내용 |
|---|---|
| 입력 | 없음 |
| 출력 | 시료 ID, 시료명, 평균 생산시간, 수율, 현재 재고 수량 전체 목록 |
| 예외 | 등록된 시료 없음 → "등록된 시료가 없습니다" 안내 후 메뉴 복귀 |

#### 4.2.3 시료 검색

검색 항목을 먼저 선택한 뒤 검색어/값을 입력한다.

| 검색 항목 | 매칭 방식 |
|---|---|
| 시료 ID | 부분 일치 |
| 시료명 | 부분 일치 |
| 평균 생산시간 | 수학적 동치 (0.9 = 0.90) |
| 수율 | 수학적 동치 (0.9 = 0.90) |

| 항목 | 내용 |
|---|---|
| 입력 | 검색 항목 선택 → 검색어 또는 수치 입력 |
| 출력 | 조건에 일치하는 시료 목록 (4.2.2 출력 형식과 동일) |
| 예외 | 검색 결과 없음 → "검색 결과가 없습니다" 안내 후 메뉴 복귀 |
| 예외 | 수치 항목에 숫자 외 입력 → 오류 메시지 출력 후 복귀 |

### 4.3 시료 주문

고객이 시료를 요청하면 주문 담당자가 시스템에 주문을 등록한다.

#### 4.3.1 주문 등록

| 항목 | 내용 |
|---|---|
| 입력 | 시료 ID, 고객명, 주문 수량 |

| 출력 | 발급된 주문 ID 및 등록 완료 메시지 |

#### 4.3.2 주문 목록 조회

| 항목 | 내용 |
|---|---|
| 입력 | 없음 (전체 조회) |
| 출력 | 주문 ID, 고객명, 시료명 (시료 ID), 주문 수량, 현재 상태, 등록 일시 |
| 예외 | 등록된 주문 없음 → "주문 내역이 없습니다" 안내 후 메뉴 복귀 |

### 4.4 주문 승인 / 거절

접수된 `RESERVED` 상태 주문 목록을 확인하고 승인 또는 거절을 처리한다.

#### 4.4.1 접수 주문 목록 조회

| 항목 | 내용 |
|---|---|
| 입력 | 없음 |
| 출력 | `RESERVED` 상태 주문의 주문 ID, 고객명, 시료명, 주문 수량, 등록 일시 |
| 예외 | 접수 주문 없음 → "대기 중인 주문이 없습니다" 안내 후 메뉴 복귀 |

#### 4.4.2 거절 주문 목록 조회

| 항목 | 내용 |
|---|---|
| 입력 | 없음 |
| 출력 | `REJECTED` 상태 주문의 주문 ID, 고객명, 시료명, 주문 수량, 등록 일시 |
| 예외 | 거절된 주문 없음 → "거절된 주문이 없습니다" 안내 후 메뉴 복귀 |

#### 4.4.3 주문 승인

| 항목 | 내용 |
|---|---|
| 입력 | 주문 ID |
| 처리 전제 | 주문 상태가 `RESERVED`이어야 함 |
| 재고 충분 시 | 재고 차감 → 상태 `CONFIRMED` 전이 |
| 재고 부족 시 | 생산 라인 대기열에 등록 → 상태 `PRODUCING` 전이 |
| 출력 | 전이된 상태 및 처리 결과 메시지 |
| 예외 | 대기 중인 주문 없음 → 오류 메시지 출력 후 메뉴 복귀 |
| 예외 | 존재하지 않는 주문 ID → 오류 메시지 출력 |
| 예외 | `RESERVED` 이외 상태 → `InvalidOrderStateTransitionException` |

#### 4.4.4 주문 거절

| 항목 | 내용 |
|---|---|
| 입력 | 주문 ID |
| 처리 전제 | 주문 상태가 `RESERVED`이어야 함 |
| 처리 결과 | 상태 `REJECTED` 전이 |
| 출력 | 거절 완료 메시지 |
| 예외 | 대기 중인 주문 없음 → 오류 메시지 출력 후 메뉴 복귀 |
| 예외 | `RESERVED` 이외 상태 → `InvalidOrderStateTransitionException` |

### 4.5 모니터링

담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 주문 현황과 재고 현황을 함께 제공한다.

#### 4.5.1 주문량 확인

| 항목 | 내용 |
|---|---|
| 입력 | 없음 |
| 출력 | `RESERVED`, `PRODUCING`, `CONFIRMED`, `RELEASE` 상태별 주문 목록 |
| 표시 정보 | 주문 ID, 고객명, 시료명 (시료 ID), 주문 수량 |
| 제외 | `REJECTED` 상태 주문은 표시하지 않음 |

**상태별 글자 색상 (ANSI)**

| 상태 | 색상 |
|---|---|
| `RESERVED` | 하늘색 (Bright Cyan) |
| `PRODUCING` | 주황색 (256색 #214) |
| `CONFIRMED` | 연두색 (Bright Green) |
| `RELEASE` | 연보라색 (Bright Magenta) |

#### 4.5.2 재고량 확인

| 항목 | 내용 |
|---|---|
| 입력 | 없음 |
| 출력 | 시료별 현재 재고 수량 및 재고 상태 |

**재고 상태 기준 및 글자 색상**

| 상태 | 조건 | 색상 |
|---|---|---|
| 여유 | 현재 재고 수량 ≥ `RESERVED` + `PRODUCING` 상태 주문의 총 요청 수량 | 연두색 (Bright Green) |
| 부족 | 현재 재고 수량 > 0 이면서 요청 수량에 미달 | 주황색 (256색 #214) |
| 고갈 | 현재 재고 수량 = 0 | 빨강색 (Bright Red) |

### 4.6 생산 라인

주문 승인 시 재고가 부족한 경우 생산 라인에 작업이 등록된다. 생산 라인은 FIFO 방식으로 작업을 순차 처리한다.

**생산량 및 생산 시간 산정 공식**

```
실 생산량     = ceil(부족분 / (수율 × 0.9))
총 생산 시간  = 평균 생산시간 × 실 생산량  [단위: min]
```

> 수율에 0.9를 곱하는 것은 공정 오차를 고려한 안전 마진이다.

#### 4.6.1 생산 현황 조회

| 항목 | 내용 |
|---|---|
| 입력 | 없음 |
| 출력 | 현재 생산 중인 주문의 주문번호, 시료명, 시료 ID, 주문량, 재고 수량, 부족 수량, 실생산량, 소요시간(min), 진행률(%) |
| 진행률 계산 | `min(100%, 경과 시간(ms) / (소요시간(min) × 60,000ms) × 100)` — 생산 시작 시각(`startedAt`) 기준 밀리초 단위 계산 |
| 자동 완료 | 진행률이 100%에 도달하면 생산 완료를 자동 처리하여 재고를 즉시 업데이트하고, 다음 대기 작업의 생산을 시작 |
| 예외 | 현재 생산 중인 작업 없음 → "생산 중인 작업이 없습니다" 안내 |

#### 4.6.2 대기 주문 확인

| 항목 | 내용 |
|---|---|
| 입력 | 없음 |
| 출력 | 생산 큐에 대기 중인 작업 목록 (FIFO 순서, 주문 ID, 시료명, 실생산량, 예상 소요시간, 접수시각) |
| 예외 | 대기 작업 없음 → "대기 중인 작업이 없습니다" 안내 |

#### 4.6.3 생산 완료 처리 (자동)

| 항목 | 내용 |
|---|---|
| 트리거 | 생산 현황 조회 시 진행률 100% 도달 |
| 처리 결과 | 실생산량만큼 재고 반영, 상태 `CONFIRMED` 전이, 다음 대기 작업 자동 시작 |
| 출력 | "[생산 완료] 재고 자동 업데이트: N개" 메시지 출력 |
| 비고 | 수동 완료 처리 메뉴 없음 — 진행률 기반 자동 처리로 대체 |

### 4.7 출고 처리

`CONFIRMED` 상태 주문에 대해 출고를 실행하여 시료를 고객에게 전달한다.

#### 4.7.1 출고 대기 주문 목록 조회

| 항목 | 내용 |
|---|---|
| 입력 | 없음 |
| 출력 | `CONFIRMED` 상태 주문의 주문 ID, 고객명, 시료명, 주문 수량 |
| 예외 | 출고 대기 주문 없음 → "출고 대기 중인 주문이 없습니다" 안내 |

#### 4.7.2 출고 실행

| 항목 | 내용 |
|---|---|
| 입력 | 주문 ID |
| 처리 전제 | 주문 상태가 `CONFIRMED`이어야 함 |
| 처리 결과 | 상태 `RELEASE` 전이 |
| 출력 | 출고 완료 메시지 |
| 예외 | 존재하지 않는 주문 ID → 오류 메시지 출력 |
| 예외 | `CONFIRMED` 이외 상태 → `InvalidOrderStateTransitionException` |
